# Workflow : weekly-maintenance-report

## Trigger
- Schedule : every Monday 9am UTC

## Content (digest pour owner)

- Bugs resolus la semaine
- Bugs encore ouverts (top 5 P0/P1)
- Coûts de la semaine vs budget
- Incidents
- Renovate PRs en attente
- Backup status (7j)
- Security score (current vs last week)
- Data quality (sellable %, opt-out %, etc.)

## Output
- Email markdown rapport
- Slack message résumé
